## 1. 项目概述与核心哲学

### 1.1 项目目标
开发一款**本地化运行**的A股分析工具，旨在通过多智能体（Multi-Agent）辩论机制辅助投资决策。系统不追求高频交易，专注于：
- **80%仓位**：月线级别选股，执行长期价值投资策略。
- **20%仓位**：日线级别择时，执行波段交易策略。

### 1.2 核心决策机制：“三智库辩论赛”
系统不依赖单一模型或简单加权打分，而是引入**模拟投资委员会**机制：
1. **三位委员（Agent）**：价值派、技术派、宏观风控派，各自独立分析同一只标的。
2. **公开辩论**：针对彼此逻辑漏洞进行2轮互相质询。
3. **最终投票**：由“裁判长Agent”根据辩论记录裁决，必须**全票通过**方可生成交易指令。

### 1.3 合规与安全底线
- **纯模拟盘**：在未经过至少3个月模拟验证前，绝不接入实盘接口。
- **人工最终确认**：所有AI生成的订单，必须通过前端页面点击“人工确认”按钮触发，**严禁全自动下单**。

4. 核心实现细节（针对需求定制）
4.1 多Agent辩论与投票机制（LangGraph实现逻辑）
在LangGraph中定义如下节点流转逻辑（非代码，仅为状态机设计）：

广播节点：将当前候选股票池（最多3只）的标准化数据包，同时分发给3个Agent。

独立思考节点：每个Agent基于自身Prompt生成 {观点, 证据逻辑, 建议仓位%, 置信度(0-100)}。

辩论节点（2轮）：

第一轮：Agent A的观点传给B，B针对A的假设进行反驳，C同理。

第二轮：在首轮基础上，各自修正或坚持原始观点。

最终裁决节点：

裁判长Agent根据辩论记录输出 final_verdict。

硬性规则：若风险官Agent的置信度 < 60%，无论价值派如何看好，直接一票否决。

4.2 L5 自动复盘与策略固化（自进化机制）
每日收盘后（15:30）自动触发复盘Agent，流程如下：

输入：当日成交记录、大盘涨跌幅、买卖点截图（存为日志）。

分析：调用LLM进行“反事实推理”（例如：如果今天没有买入XX，组合收益率会相差多少？）。

输出：生成结构化的 《策略进化日志_YYYY-MM-DD.MD》，明确包含：

今日正确决策归因。

今日错误决策归因。

针对策略YAML文件的参数微调建议（如：建议将止盈线从8%上调至9.5%）。

固化：通过Git自动提交该文档，形成可追溯的经验库。

4.3 前端“手动修改策略”的实现方案（低代码设计）
在前端设计 “策略参数工厂” 界面，所有修改实时映射到后端config.yaml：

策略模块	可调参数（交互形式）	生效逻辑
价值选股阈值	PE ≤ [15]（滑块）；股息率 ≥ [2.5]%（输入框）	点击“保存”后，更新RAG检索的SQL条件。
波段止盈止损	止盈 [+8]%；止损 [-5]%；持仓周期 ≤ [10] 天	点击“保存”后，更新风控Agent的系统Prompt元数据。
仓位管理系数	单票最大仓位 [15]%；大盘>3200点仓位系数 [0.6]	修改后直接注入PositionSizing函数变量。
5. 前端设计原型（关键界面参考）
5.1 页面一：投资决策大厅（Dashboard）
左侧面板：展示当前持仓（股票代码、成本价、浮盈亏比例）。

7. A股特供版关键风险应对
A股具有极强的政策性与规则特殊性，必须在底层硬编码处理：

ST/退市风险：在数据清洗层（data_pipeline.py）硬编码过滤，若is_st = 1或status='停牌'，直接剔除候选池，禁止进入辩论环节。

涨跌停限制：模拟执行层需校验当前价格是否为涨/跌停价。若是，则自动撤销该订单并向前端推送提示“该标的一字封板，无法成交”。

T+1与资金隔离：

价值仓（80%）：买入后当日禁止卖出，系统自动锁定。

波段仓（20%）：允许T+1操作，且需在清算时严格区分两笔资金池，防止计算混乱。

8. 免费数据源落地实施方案
由于要求先跑通本地流程，数据获取按以下优先级实施：

行情数据（日频）：使用 akshare 的 stock_zh_a_hist 接口（免费、无调用次数限制，但需注意网络超时重试）。

基本面数据（PE/PB/ROE）：使用 akshare 的 stock_financial_abstract 接口，按季度获取。

新闻情绪（简化版）：

使用 RSSHub 或 akshare 的 stock_news_em 接口抓取“财联社”或“东方财富”的公开热榜标题。

利用本地 BAAI/bge-large-zh Embedding模型将标题转为向量，存入Milvus进行简单的热词情绪打分，不依赖任何付费财经API。

9. 开发者启动指南（PM建议）
在编写任何代码前，强烈建议按以下步骤降低试错成本：

Step 1（单点测试）：不要一上来就扫描全市场。先写一个脚本，仅对 000001.SZ（平安银行）运行三Agent辩论流程，观察输出是否合理。

Step 2（录制回放）：记录每一次辩论的全文日志。这些文本既是调试依据，也是未来微调本地Qwen模型的绝佳训练语料。

Step 3（心态管理）：严格区分“模拟盈利”与“实盘能力”。务必在本地连续跑通3个月，且日频逻辑无任何KeyError或网络超时后，再考虑以极低仓位（如1万元）接入实盘观察滑点影响。
